JavaScript functions to add to index.html

loadConvictionStats(signal)

— called by liveRefresh() and switchPage('convictions')

Calls GET /api/convictions/stats (no ticker filter). Writes the seven aggregate stat cards in the top bar. Also writes the pending count badge in the sidebar nav. Triggers a visual flash on the pending count badge if it changed since last call.

DOM IDs written:
  cl-stat-ml-30, cl-stat-ml-60, cl-stat-ml-90
  cl-stat-sec, cl-stat-spy
  cl-stat-total, cl-stat-pending
  nav-pending-count (sidebar badge)

loadConvictionLedger(signal, page=1, ticker=null, signalType=null, window=null)

Calls GET /api/convictions/ledger?page=N&ticker=X. Renders the conviction history table, including state-specific cell rendering for confirmed (green ✓), contradicted (red ✗), and pending (amber timestamp) cells. Also handles regime breakdown rendering from the stats payload. Wired to filter dropdowns via onchange handlers that re-call this function.

DOM IDs written:
  cl-table-body (tbody of conviction table)
  cl-pagination (page 1 of N display)
  cl-regime-low, cl-regime-mid, cl-regime-high

loadTickerConvictionBadge(ticker, containerEl)

Calls GET /api/convictions/stats?ticker=NVDA. Renders the 1-line inline compact view footer into containerEl — which is the last child of the generated memo/signal output block. If the stats response has zero resolved convictions for that ticker, renders the "first call" variant instead. Called by the memo generation renderer and the signal card renderer immediately after their content is injected into the DOM.

No fixed DOM IDs — appends to dynamic containerEl passed by caller.
Caller pattern: loadTickerConvictionBadge(ticker, memoOutputEl)

resolveConvictionsPoll(signal)

— called by liveRefresh() in background

Calls GET /api/convictions/check. Silent background call — no UI spinner. If response resolved > 0, triggers a toast notification: "X convictions resolved — accuracy updated" and re-runs loadConvictionStats(). If the user is currently on the convictions page, also re-runs loadConvictionLedger() to refresh the table live. Never blocks the rest of liveRefresh.

No DOM IDs of its own.
Calls: loadConvictionStats(), optionally loadConvictionLedger(), showToast()

liveRefresh() integration — two additions

Two lines are added to the existing liveRefresh() Promise.all array. Both are fire-and-forget from the dashboard's perspective — they run in parallel with the other fetches and never cause the refresh to stall even if the convictions API is slow.

// existing liveRefresh() Promise.all — additions shown with arrows
await Promise.all([
  loadDashboardMarket(sig),
  loadDashboardSignals(sig),
  loadDashboardAlerts(sig),
  loadPortfolioPage(sig),
  loadDashboardModelAccuracy(sig),
  loadWatchlistPrices(sig),
  loadConvictionStats(sig),       // ← NEW: writes sidebar badge + stat cards
  resolveConvictionsPoll(sig),   // ← NEW: checks for due lookbacks silently
]);

Additionally, the loaders object in switchPage() gains one new entry:

convictions: () => Promise.all([loadConvictionStats(sig), loadConvictionLedger(sig)]),

Visual state reference — table cell rendering rules

Confirmed — direction correct

✓ +11.4%

Green background, checkmark, actual return. Tooltip: "[ticker] was [BULLISH/BEARISH], price moved [+/-X%]"

Contradicted — direction wrong

✗ +6.3%

Red background, X mark, actual return. Row gets a faint red row background rgba(255,90,122,0.02) to make errors scannable.

Pending — not yet due

⏳ Apr 30

Amber for the nearest upcoming window. Muted gray text for farther windows. Due date shown in "Mon DD" format.
