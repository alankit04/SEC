Backend — one new file, three new API endpoints, two integration points

New file: backend/conviction_store.py

Single responsibility: read, write, and resolve convictions. Nothing else touches the JSONL files directly.

write_conviction()

Called when any research output is generated. Accepts the typed outputs from market-analyst, sec-researcher, ml-signals agents. Builds the conviction object, appends to convictions.jsonl.

write_conviction(
  ticker: str,
  ml: MLSignalOutput,
  sec: SECResearcherOutput,
  signal_view: str,
  source: str
) → str # returns conviction_id

check_pending()

Called by liveRefresh via API. Reads all convictions, joins with resolutions to find unresolved lookback windows whose due dates have passed. For each: fetches price via yfinance, computes result, appends to resolutions.jsonl.

check_pending() → {
  "resolved": int,
  "still_pending": int,
  "errors": list[str]
}

get_accuracy_stats()

Reads both JSONL files, joins by conviction_id, computes accuracy scores. Returns stats by signal type, by lookback window, and by ticker. This powers both the ledger page and the inline compact view.

get_accuracy_stats(
  ticker: str | None,
  lookback: str | None
) → AccuracyStats

get_ledger()

Returns full conviction history for the ledger page: all convictions with their resolved/pending status per lookback window. Sorted by date descending. Paginated (50 per page).

get_ledger(
  page: int = 1,
  ticker: str | None
) → LedgerPage

Three new API endpoints in raphi_server.py

POST /api/convictions
Called internally when a memo or signal is generated. Writes a new conviction. Rate limited: 60/min (data tier). Returns conviction_id.
GET /api/convictions/check
Called by liveRefresh every cycle. Resolves all elapsed pending lookbacks. Idempotent — safe to call repeatedly. Returns {resolved, still_pending, errors}.
GET /api/convictions/stats?ticker=NVDA
Returns accuracy stats. ticker param is optional — omit for portfolio-wide stats. Powers both the ledger page aggregate view and the inline compact accuracy line on each memo.
GET /api/convictions/ledger?page=1&ticker=NVDA
Full conviction history for the ledger page. Paginated at 50 per page. ticker param optional for filtering.

Two integration points in existing code

1. a2a_executor_v2.py — on research completion

After the memo-synthesizer returns a validated output, call write_conviction() with the typed agent outputs. This is the only place convictions are created — not from chat, not from the dashboard, only from completed research outputs.

# After schema validation passes:
conviction_id = write_conviction(
  ticker, ml_out, sec_out,
  memo.signal_view, "memo"
)

Error handling rules

yfinance fails during check: Skip that conviction this cycle. Nothing written. Retry next liveRefresh.
EDGAR data not yet updated: SEC check skipped. Only fires when new num.txt data exists for that ticker's next expected period.
Conviction already has a resolution for that lookback: Skip. Resolution JSONL is deduplicated by (conviction_id, lookback) on read.
File write error: Log to Sentry. Return error in check response. UI shows "ledger unavailable" gracefully.
