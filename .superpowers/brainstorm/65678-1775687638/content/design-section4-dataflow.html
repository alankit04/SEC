Read & Join Phase
① Read convictions.jsonl
Build dict: id → {ticker, ml.direction, entry_price, lookbacks_due}
② Read resolutions.jsonl
Build set: {conviction_id + lookback} already resolved — skip these
③ Filter: past-due & unresolved
For each conviction × {30d,60d,90d}: if today ≥ lookbacks_due[window] AND not in resolved set → enqueue
yfinance price fetch
yf.Ticker(ticker).history(period="5d")
Use most recent market close ≥ due_date
Also fetch SPY for relative return computation
→
Result Computation
price_return = (close - entry_price) / entry_price × 100
CONFIRMED: LONG & price_return > 0, or SHORT & price_return < 0
CONTRADICTED: opposite direction
vs_spy_pct = price_return − SPY return over same window
vix_at_check: fetched from ^VIX history
→
Append resolution
resolutions.jsonl ← one line
{conviction_id, lookback, ml_result,
price_at_check, vs_entry_pct, vs_spy_pct, vix_at_check}
Edge Cases
yfinance fails / timeout: skip this resolution, add to errors[], still_pending++. Retried on next liveRefresh cycle. No partial write.
Already resolved: resolved set lookup (O(1)) prevents duplicate. Idempotent — safe to run multiple times.
Market holiday: history(period="5d") returns last available close. If no data in 5d window → treated as yfinance failure, deferred.

Scenario C — SEC Filing Resolution (data-triggered, not time-triggered)

Trigger: same liveRefresh()
check_pending() also handles SEC window
lookback key = "sec", not a fixed day offset
→
Condition: new filing has arrived
conviction.sec.next_filing_due is stored at creation time (from @sec-researcher).
check_pending() logic: today ≥ next_filing_due AND "sec" not in resolved set
→ attempt SEC resolution for this conviction
EDGAR data fetch via sec_data.py
sec.company_financials(ticker) → reads local XBRL num.txt files
Filters to quarters filed AFTER conviction date
Pulls "Revenues" tag (or fallback synonyms) for latest available quarter
Returns: actual_revenue, period_of_report
→
Trend verification logic
revenue_delta_pct = (actual − latest_revenue) / latest_revenue × 100
CONFIRMED: trend="accelerating" & delta > +3%, or trend="decelerating" & delta < −3%
CONTRADICTED: direction mismatch beyond threshold
INCONCLUSIVE: |delta| ≤ 3% (noise band) — excluded from accuracy denominator
→
Append SEC resolution
resolutions.jsonl ←
{conviction_id, lookback:"sec",
sec_result, actual_revenue,
revenue_delta_pct, resolved_date}
Edge cases: No new filing yet (today < next_filing_due): skip, still_pending++. EDGAR data missing for ticker: error[], deferred. Filing arrived but revenue tag absent in XBRL: INCONCLUSIVE written, excluded from denominator. sec.next_filing_due was null at creation (partial_memo): SEC resolution never attempted for this conviction.

Scenario D — Accuracy Stats Computation

① Read convictions.jsonl
Build lookup: conviction_id → {ticker, ml.direction, sec.trend, signal_view, source, date}
② Read resolutions.jsonl
Group by conviction_id: dict[id] = list[resolution]. Each resolution has lookback key + result value.
③ Join & Compute
For each conviction: join its resolutions. For each resolved lookback: tally CONFIRMED or CONTRADICTED. INCONCLUSIVE and still-pending: excluded from denominator entirely.
④ Aggregate Dimensions
accuracy = CONFIRMED / (CONFIRMED + CONTRADICTED)
Slices: by signal_type (ml/sec), by lookback (30d/60d/90d/sec), by ticker, overall
Denominator rule: A conviction with zero resolutions is "pending" — never enters the accuracy calculation. A conviction with some resolutions only contributes for the windows that have resolved. Example: a 15-day-old conviction contributes nothing (all 3 ML windows still future), a 35-day-old conviction contributes its 30d result only.
