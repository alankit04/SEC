Data Model — the two JSONL files and what lives in each

convictions.jsonl — write once, never mutate

One line appended every time RAPHI generates a research output (memo, signal, or chat with a ticker conclusion). Immutable.

{
  "id": "cvx-20260408-NVDA-a3f",
  "ticker": "NVDA",
  "date": "2026-04-08T14:32:11Z",
  "entry_price": 890.50,
  "ml": {
    "direction": "LONG",
    "probability": 0.71,
    "model_version": "xgb_v2.1"
  },
  "sec": {
    "trend": "accelerating",
    "latest_revenue": 44200000000,
    "quarters_used": 8,
    "next_filing_due": "2026-07-15"
  },
  "signal_view": "Positive",
  "conviction": "MEDIUM",
  "source": "memo",
  "lookbacks_due": {
    "30d": "2026-05-08",
    "60d": "2026-06-07",
    "90d": "2026-07-07"
  }
}

resolutions.jsonl — append on each check

One line per resolved lookback window. A single conviction generates up to 4 resolution lines: 30d ML, 60d ML, 90d ML, and SEC filing. Each is independent.

// ML 30-day check
{
  "conviction_id": "cvx-20260408-NVDA-a3f",
  "lookback": "30d",
  "resolved_date": "2026-05-08",
  "ml_result": "CONFIRMED",
  "price_at_check": 962.30,
  "vs_entry_pct": 8.1,
  "vs_spy_pct": 6.8,
  "vix_at_check": 16.2
}

// SEC filing check (when 10-Q arrives)
{
  "conviction_id": "cvx-20260408-NVDA-a3f",
  "lookback": "sec",
  "resolved_date": "2026-07-16",
  "sec_result": "CONFIRMED",
  "actual_revenue": 49800000000,
  "revenue_delta_pct": 12.7
}

Key design rules

IDs: cvx-{date}-{ticker}-{random3} — human-readable, unique, sortable by date

convictions.jsonl is append-only — never read to resolve. Only read for display.

source field tracks what generated the conviction: "memo" · "signal_query" · "chat"

resolutions.jsonl is also append-only — if a check fails (API error), nothing is written. Retry next cycle.

vix_at_check stored on every resolution — enables future regime analysis without reprocessing.

SEC check only fires when new EDGAR data arrives for that ticker — not on a fixed date.
