What does "proved true" mean?

Each of the four tracked outputs needs a clear, automatable definition of correctness. Some are obvious. One is not. Pick the definition set that feels right.

ML Signal — unambiguous

Price direction at 30 / 60 / 90 days

If signal said LONG and price is higher at the checkpoint → CONFIRMED.
If signal said SHORT and price is lower → CONFIRMED.
Measured vs closing price on conviction date.

NVDA LONG @ $890 → 30d: $962 → ✅ +8.1%

SEC Trend — unambiguous

Next quarter XBRL confirms or contradicts

If verdict said "accelerating" and next 10-Q revenue is higher QoQ → CONFIRMED.
If verdict said "decelerating" and next 10-Q is lower → CONFIRMED.
Checked automatically when new EDGAR data arrives.

NVDA Revenue: Accelerating → Q2 filing: +12% QoQ → ✅

Sentiment — needs a choice

Three options — pick one

S1

5-day price reaction to news

Positive sentiment → price up 5 days later. Fast feedback.

S2

Vs. sector return at 30 days

Positive sentiment → stock outperforms its sector peers. Removes market noise.

S3

Don't score sentiment separately

Sentiment is an input to the Signal View. Score the Signal View only. Sentiment accuracy is implicit.

Signal View — the composite

Risk-adjusted return vs S&P 500

The final Positive / Negative / Neutral verdict is checked against the stock's return relative to SPY at 30/60/90 days.

Positive → stock outperforms SPY → CONFIRMED.
Negative → stock underperforms SPY → CONFIRMED.
Neutral → within ±2% of SPY → CONFIRMED.

NVDA Positive → +8.1% vs SPY +1.3% → ✅ outperformed

My recommendation on sentiment

S3 — Don't score sentiment separately. VADER is the weakest signal in RAPHI. Giving it a standalone accuracy score risks surfacing that it's ~50% accurate, which erodes trust in the rest of the ledger. Sentiment feeds into the Signal View, which already gets scored vs. SPY. That's sufficient. Add standalone sentiment scoring in a later version once VADER is replaced with a better model.
